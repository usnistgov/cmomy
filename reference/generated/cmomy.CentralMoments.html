cmomy.CentralMoments#

class cmomy.CentralMoments(data, mom_ndim=1)[source]#

Bases: CentralMomentsABC[NDArray[MyDType]]

Wrapper to calculate central moments.

Base data has the form

\[\begin{split}{\rm data}[..., i, j] = \begin{cases} \text{weight} & i = j = 0 \\ \langle x \rangle & i = 1, j = 0 \\ \langle (x - \langle x \rangle^i) (y - \langle y \rangle^j) \rangle & i + j > 0 \end{cases}\end{split}\]
Parameters:

  • data (numpy.ndarray) – Moment collection array

  • mom_ndim ({1, 2}) – Value indicates if moments (mom_ndim = 1) or comoments (mom_ndim=2).

classmethod zeros(mom=None, mom_ndim=None, val_shape=None, shape=None, dtype=None, zeros_kws=None, **kws)[source]#

Create a new base object.

Parameters:

  • mom (int or tuple of int) – Order or moments. If integer or length one tuple, then moments are for a single variable. If length 2 tuple, then comoments of two variables

  • mom_ndim ({1, 2}) – Value indicates if moments (mom_ndim = 1) or comoments (mom_ndim=2).

  • val_shape (tuple, optional) – Shape of values part of data. That is, the non-moment dimensions.

  • shape (tuple, optional) – Total shape. shape = val_shape + tuple(m+1 for m in mom)

  • dtype (dtype, optional) – Optional dtype for output data.

  • zeros_kws (mapping, optional) – Optional parameters to numpy.zeros()

  • **kws – extra arguments to from_data()

Returns:

output (CentralMoments) – New instance with zero values.

Notes

The resulting total shape of data is shape + (mom + 1). Must specify either mom or mom_ndim

raw[…, i, j] = <x**i y**j>. raw[…, 0, 0] = weight

Parameters:

  • cls – The description is missing.

  • raw (ndarray) – Raw moment array.

  • mom_ndim ({1, 2}) – Value indicates if moments (mom_ndim = 1) or comoments (mom_ndim=2).

  • mom (int or tuple of int) – Order or moments. If integer or length one tuple, then moments are for a single variable. If length 2 tuple, then comoments of two variables

  • val_shape (tuple, optional) – Shape of values part of data. That is, the non-moment dimensions.

  • dtype (dtype, optional) – Optional dtype for output data.

  • convert_kws (mapping) – Extra arguments to to_central_moments() or to_central_comoments()

  • **kws – Extra arguments to from_data()

Returns:

output (CentralMoments)

See also

to_raw, rmom, to_central_moments, to_central_comoments

Notes

Weights are taken from raw[...,0, 0]. Using raw moments can result in numerical issues, especially for higher moments. Use with care.

Examples

>>> np.random.seed(0)
>>> x = np.random.rand(10)
>>> raw_x = (x[:, None] ** np.arange(5)).mean(axis=0)

>>> dx_raw = CentralMoments.from_raw(raw_x, mom_ndim=1)
>>> dx_raw.mean()
0.6157662833145425
>>> dx_raw.cmom()
array([1.    , 0.    , 0.034 , 0.0038, 0.0026])

Which is equivalent to creating raw moments from values >>> dx_cen = CentralMoments.from_vals(x, axis=0, mom=4) >>> dx_cen.mean() 0.6157662833145425 >>> dx_cen.cmom() array([1. , 0. , 0.034 , 0.0038, 0.0026])

But note that calculating using from_raw can lead to numerical issues. For example

>>> y = x + 10000
>>> raw_y = (y[:, None] ** np.arange(5)).mean(axis=0)
>>> dy_raw = CentralMoments.from_raw(raw_y, mom_ndim=1)
>>> dy_raw.mean() - 10000
0.6157662833156792
>>> dy_raw.cmom()  # note that these don't match dx_raw, which they should
array([ 1.0000e+00,  0.0000e+00,  3.4031e-02,  4.7744e-03, -1.8350e+01])

>>> dy_cen = CentralMoments.from_vals(y, axis=0, mom=4)
>>> dy_cen.mean() - 10000
0.6157662833156792
>>> dy_cen.cmom()  # this matches above
array([1.    , 0.    , 0.034 , 0.0038, 0.0026])
